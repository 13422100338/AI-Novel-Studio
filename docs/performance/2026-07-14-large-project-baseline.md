# 大型项目性能诊断基线（2026-07-14）

## 测试范围

- 测试来源：用户提供的桌面 `X` 项目。
- 安全措施：仅复制 `project.json`、`project.sqlite3`、`manuscript` 与 `.ai_pipeline` 到临时目录；所有运行时测试均在副本执行。
- 渲染环境：`QT_QPA_PLATFORM=offscreen`。
- 数据库读取：原项目只以 SQLite 只读模式统计，未修改稿件、数据库、记忆、备份或导出。

## 已确认热点

1. 正文每次 `textChanged` 都立即复制并扫描全文统计字数。
2. 草稿预览期间正文框与隐藏草稿框逐字符执行双向全文读取与覆盖。
3. 正文流式输出同时写入两个编辑器，每个小 chunk 都触发布局更新。
4. 聊天气泡每个小 chunk 都执行完整字符串读取、拼接、`QLabel.setText` 与滚动。
5. 切换章节同步刷新全部人物状态；39 个人物分别查询时间线。
6. 保存章节后重建整棵卷章树。
7. 章节树启用动画、非统一行高，批量重建时未暂停 updates。
8. 记忆库一次性加载 764 条完整记录，首屏超过 800 ms。

## 第一批低风险修改范围

1. 字数统计增加 250 ms 单次防抖。
2. 正文流式 chunk 使用 40 ms 缓冲，只更新一个可见草稿数据源。
3. 聊天流式 chunk 使用 40 ms 缓冲，完成/错误前强制刷新。
4. 关闭章节树动画，启用统一行高，批量更新时暂停重绘。
5. 保存或采用当前章节后只更新当前树节点，不重建全树。
